- Тип: финансовый учёт и аналитика
- Стек: Python

- Комментарии: на русском (только сложные места)

- Фреймворк: pytest
- Тесты обязательны для: всего

## Строка статуса
При каждом ответе показывай в начале (кратко, одной строкой):
- Текущую модель
- Рабочую директорию
- Использованный контекст (токены)
- Лимиты (если есть)

Пример: `[llava:7b | C:\project | 1.2k/8k токенов]`

## Процесс работы
1. Сначала изучи код и существующие конвенции
2. Предлагай изменения, но не применяй их без подтверждения
3. Не создавай лишних файлов (README, docs) без явной просьбы
4. Не запускай git commit/push без моего разрешения
5. После изменений запускай линтер и тесты

## Коммуникация
- Язык общения: русский / английский
- Ответы: кратко, по делу
- Код должен говорить сам за себя — минимум комментариев в коде

## Сохранение переписки
- Всегда сохраняй полную переписку сессии в файл перед завершением работы
- Формат имени: `session--YYYY-MM-DD--<проект>.md`, где `<проект>` — имя текущей папки проекта (например, `books`, `finance-card`)
- Формат содержимого:
  ```markdown
  # Сессия: YYYY-MM-DD
  ## Проект: <название>

  ### User: <сообщение>

  ### Assistant: <ответ>
  ```
- Сохранять в корень текущего проекта

## Инструменты и среда
- **Нейминг файлов:** рабочая формула — `[дата-опционально]--[тема]--[тип].md`
  - Примеры: `2026-04-21--sales--weekly-report.md`, `client-onboarding--checklist.md`
  - Дата указывается в формате ГГГГ-ММ-ДД, если важна хронология
  - Разделитель между частями — двойной дефис `--`
  - Расширение — `.md` (Markdown)
