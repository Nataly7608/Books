# Урок 1: Настройка и работа с AI-ассистентом (opencode)

**Дата:** 05.06.2026
**Проект:** pa-finance (финансовый учёт и аналитика)
**Стек:** Python
**Инструмент:** [opencode](https://opencode.ai) — CLI-агент для разработки

## Что сделано

1. **Инициализация проекта**
   - Создана корневая структура `pa-finance`
   - Установлен и настроен opencode

2. **Конфигурация opencode** (`opencode.json`)
   - Подключена модель `ollama/llava:7b` (локальный запуск через Ollama)
   - Настроен Provider: `http://localhost:11434/api`
   - Подключены инструкции из `AGENTS.md`
   - Включена автоматическая компактизация контекста

3. **Инструкции для ассистента** (`AGENTS.md`)
   - Описан профиль проекта (Python, финансовый учёт)
   - Заданы правила стиля кода: смешанный ООП + функциональный, минимум комментариев
   - Тестирование: pytest (обязательно для всего)
   - Формат ответа: строка статуса с моделью, директорией и токенами
   - Процесс работы: изучение кода → предложение → подтверждение → без лишних commit'ов

4. **Настройка TUI** (`tui.json`)
   - Тема: Catppuccin
   - Включена поддержка мыши
   - Назначены хоткеи: `<leader>s` — статус, `<leader>d` — показать thinking

5. **Первый сеанс работы**
   - Изучена текущая структура проекта
   - Создан данный файл урока
   - Отработана механика: вопрос → ответ → действие через tools

## Выводы

- opencode удобен как ассистент внутри терминала: не требует переключения контекста
- Локальная модель (llava:7b через Ollama) даёт достаточную скорость для работы с небольшими проектами
- AGENTS.md — эффективный способ задать поведение ассистента под конкретный проект
- Компактизация контекста (compaction.auto: true) критична для длительных сессий — без неё модель быстро теряет фокус
- TUI-режим с кастомной темой и хоткеями делает работу визуально приятнее
- Важно явно запрещать ассистенту делать commit/push без разрешения — иначе можно получить неожиданные изменения в репозитории
