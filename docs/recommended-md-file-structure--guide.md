# Рекомендуемая структура MD-файла

Шаблон для создания AGENT.md — персонального файла с инструкциями для AI-ассистента.

## Структура разделов

### Твой человек
Кто ты: профессия, сфера деятельности, уровень опыта в задачах, которые будешь давать агенту. Чем больше агент знает о тебе — тем меньше нужно объяснять каждый раз.

### Проекты
Над чем ты работаешь: проекты, направления, регулярные задачи. Что агент должен знать, чтобы понимать контекст без лишних вопросов.

### Как работаем
Как ты хочешь, чтобы агент подходил к задачам: сначала показывает план или сразу действует, когда уточняет перед стартом, когда может работать самостоятельно. Как проверяем результат: нужен ли отчёт, список изменённых файлов, ссылки на источники.

### Чего не делать
**Две части:**

1. **Границы** — что агент не делает без твоего явного разрешения: удалять файлы, переименовывать, отправлять сообщения, выходить за пределы рабочей папки.
2. **Анти-паттерны** — что тебя раздражает в ответах по умолчанию: длинные вступления, фраза "конечно!", пересказ того, что только что сделал, избыточные объяснения очевидного.

### Частые задачи
Список того, что ты делаешь регулярно, с описанием ожидаемого поведения агента для каждого. Пример: "когда прошу анализ таблицы — возвращай markdown-файл с выводами".

### Инструменты и среда
**Две части:**

1. Приложения и сервисы, которые ты используешь в работе и с которыми агент может взаимодействовать. Например: Obsidian для заметок, Notion для задач, Google Sheets для таблиц.
2. Кастомные инструменты или команды, которые доступны агенту в этом проекте: скрипты, CLI-утилиты.

### Формат ответа
Как ты хочешь видеть ответы: коротко или развёрнуто, с заголовками или без, нужен ли итоговый вывод в конце.

### Прочее
Всё остальное, что не попало под категории выше.

## Правила составления

- Пиши конкретно, не обобщай. Лучше "не удаляй файлы без подтверждения" чем "будь осторожен".
- Раздел "Чего не делать" всегда заполняй двумя частями: границы и анти-паттерны.
- Раздел "Частые задачи" заполняй только если из дампа ясно видно повторяющиеся сценарии.
- Раздел "Инструменты и среда" заполняй только приложениями и инструментами, которые явно упомянуты.
- Если из дампа что-то неясно — лучше уточни, чем придумывай.
- Не добавляй разделы, которых нет в шаблоне.
- Готовый файл верни как код-блок с разметкой markdown.
